👨‍💻 Code-Together 11: Probing Hash Tables

🎯 Overview

An in-class code-together activity introducing linear probing (closed hashing / open addressing) — the second collision resolution strategy after CT10’s separate chaining. Students build a ProbingHashTable that stores entries directly in a flat array, resolves collisions by scanning forward one slot at a time, and uses tombstone deletion to preserve probe sequences. The activity covers load factor monitoring with a lower threshold (0.75 vs chaining’s 1.0) and resizing with rehashing that clears tombstones. The scenario continues the professor’s grade book: student names are keys, integer grades are values.

📖 Learning Objectives

By completing this activity, you will be able to:

  • Implement linear probing (closed hashing): collisions resolved by scanning forward to the next open slot in a flat array
  • Explain why probing uses a lower load factor threshold (0.75 vs 1.0) — primary clustering makes probe sequences longer as the table fills
  • Implement tombstone deletion: removed entries are marked DELETED rather than EMPTY so probe sequences for other keys are not broken
  • Explain the three slot states (EMPTY, OCCUPIED, DELETED) and how each affects insert, search, and remove operations
  • Implement resize with tombstone cleanup: only OCCUPIED entries are rehashed, clearing all tombstones for a clean table
  • Compare probing vs chaining trade-offs: cache locality vs clustering sensitivity, flat array vs pointer chains

📚 Teaching Order

Work through the files in order:

1. HashSlot.h — The slot structure

Walk through the HashSlot struct and SlotStatus enum. Understand why three states are needed.

Section What You’ll Learn
SlotStatus enum EMPTY (never used), OCCUPIED (live entry), DELETED (tombstone) — three states that control probing behavior
HashSlot struct Key, value, and status fields; default constructor sets status to EMPTY

2. ProbingHashTable.h — Class declaration

Compare to CT10’s ChainingHashTable.h. Highlight the key differences.

Section What You’ll Learn
HashSlot* vs ChainNode** Flat array of structs vs array of linked list pointers — everything lives in the array itself
MAX_LOAD_FACTOR = 0.75 Lower threshold than chaining’s 1.0 because primary clustering worsens as the table fills
Same interface insert, search, remove, hash, load_factor, resize, print — same operations, different internals

3. ProbingHashTable.cpp — Linear probing implementation (0 TODOs)

All code is given — students follow along as the instructor walks through each method.

Section What You’ll Learn
1. Constructor Allocate HashSlot array; default constructor sets every slot to EMPTY
2. Destructor Simple delete[] table_ — no chains to walk (contrast with chaining)
3. hash() Same multiply-and-add algorithm (×31) as CT10; hash function doesn’t care about collision strategy
4. insert() Resize before inserting if load factor would exceed 0.75; probe forward with (index + 1) % capacity_ until finding EMPTY, DELETED, or matching key
5. search() Probe forward: OCCUPIED + match = found, DELETED = keep probing, EMPTY = not found
6. remove() Find the key, set status to DELETED (tombstone); explain why EMPTY would break probe chains
7. load_factor() Same formula as CT10, lower threshold; discuss clustering impact at high load factors
8. resize() Save old array, allocate new prime-sized array, rehash only OCCUPIED entries — tombstones are cleared
9. print() Display each slot: entry, [empty], or [deleted]

🔧 Troubleshooting

CMake does not configure automatically

  1. Make sure you have Desktop development with C++ workload installed in Visual Studio
  2. Go to Project → CMake Settings and ensure the configuration is correct
  3. Try Project → Delete Cache and Reconfigure
  4. Verify that CMakeLists.txt is in the root of your cloned repository

Tests fail to build

  • Google Test is fetched automatically via CMake’s FetchContent — make sure you have an internet connection during the first build
  • Do not modify header files in the include/ folder

Running the Program

  1. In Visual Studio, select ct11-probing-hash-tables.exe as the startup item from the dropdown
  2. Press Ctrl+F5 to run without debugging
  3. You should see output from each section as you complete the TODOs
Tip: Look for the // ! DISCUSSION: comments in each file — these explain the “why” behind each concept. The // ? SEE DIAGRAM: comments point you to visual aids in the images/ folder.
